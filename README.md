# About me

First of all, I'm **definitely not a professional programmer or software developer**. I only know the **basic fundamentals of programming**, just enough to read and understand how a codebase works. But in reality, I can't even build a complete project from scratch manually without the help of AI (especially ChatGPT).

To be honest...

Every project you'll find on this GitHub was first generated by **ChatGPT** (and sometimes other AI tools). I rely on AI because, honestly, it makes development much easier. I simply describe what I want, provide detailed instructions, and AI generates most of the code within minutes.

However, that doesn't mean the generated code works perfectly.

As many fellow **Certified ChatGPT Users™** probably know, AI can generate bugs, inefficient solutions, or code that doesn't quite fit the project. That's where my role comes in.

I guide the entire development process. I decide what the project should do, how it should behave, and what the final result should look like. I test everything, identify problems, and then either:

* regenerate the code,
* try another AI model,
* or manually read, understand, debug, and modify the generated code until it works as intended.

After that, I usually let AI review the updated code again, almost like having another tester or debugger.

So yes...

I'm not the one writing most of the code.

I'm the one directing how the project is built.

That's simply how I enjoy creating things. Since AI already exists, why not use it in a productive way while continuing to learn along the process?
